# Comparison Table Component

Version: 1.0

---

# Purpose

The Comparison Table Component helps customers evaluate multiple options quickly and confidently.

Its purpose is not to overwhelm the customer with information.

Its purpose is to simplify decision-making.

A well-designed comparison table reduces cognitive effort and highlights the most appropriate option for the customer.

---

# Core Philosophy

People naturally compare before they commit.

If the business does not provide a comparison, the customer will create one mentally—or compare with competitors.

The goal is to guide comparison ethically.

---

# Component Objective

The Comparison Table should:

Simplify evaluation.

Highlight differences.

Reduce confusion.

Support the recommended option.

Increase purchase confidence.

# Table Design Principles

The table should:

Be easy to scan.

Use consistent categories.

Highlight meaningful differences.

Avoid information overload.

Use no more than 6–8 comparison criteria.

# Highlighting Strategy

One option may be marked as:

Recommended

↓

Most Popular

↓

Best Value

↓

Enterprise Choice

Only highlight an option when there is a genuine reason.

---

# Decision Support

After the table, explain:

Who each option is for.

Example

Starter

Best for businesses validating an idea.

Professional

Best for growing companies that need predictable lead generation.

Enterprise

Best for organizations requiring custom workflows and dedicated support.

Help the customer self-identify.

---

# Visual Hierarchy

The recommended option should stand out subtly.

Use:

Spacing.

Icons.

Badges.

Contrast.

Avoid aggressive visual manipulation.

# Detection Signals

Use comparison tables when:

Three or more plans exist.

Feature differences matter.

Enterprise sales.

Complex pricing.

Customer confusion.

---

# AI Decision Rules

IF

Only One Offer Exists

THEN

Do not create an internal comparison table.

Consider comparing against common alternatives instead.

---

IF

Offer = Premium

THEN

Highlight:

Support.

Customization.

Implementation.

ROI.

Not simply the number of features.

---

IF

Audience = Analytical

THEN

Increase detail.

Avoid marketing language.

# Comparison Quality Checklist

Confirm

Comparison is relevant.

Criteria matter to the audience.

Differences are meaningful.

Recommended option is justified.

Easy to scan.

No unnecessary complexity.

Supports the CTA.

---

# Common Mistakes

Comparing too many criteria.

Highlighting every option equally.

Using technical language without explanation.

Creating artificial differences.

Making the cheapest option appear intentionally weak.

Ignoring customer priorities.

# Ethical Guidelines

Never manipulate comparisons.

Never misrepresent competitors.

Never invent feature differences.

Never hide important limitations.

The comparison should help the customer make a better decision—not pressure them into one.

---

# Final Rule

A comparison table should not answer:

"Which option has the most features?"

It should answer:

"Which option is the best fit for this customer?"

# Project Memory Engine

## Purpose

The Project Memory Engine maintains strategic continuity across an entire project.

Its responsibility is not to store everything.

Its responsibility is to preserve the decisions that influence future communication.

Every new deliverable should build upon previous strategic decisions rather than starting from scratch.

---

# Core Philosophy

Memory is not information.

Memory is preserved reasoning.

The engine should remember:

What was decided.

Why it was decided.

Whether the decision is still valid.

---

# Memory Hierarchy

Project Memory consists of multiple layers.

Global Memory

↓

Business Memory

↓

Brand Memory

↓

Campaign Memory

↓

Deliverable Memory

↓

Session Memory

Each layer inherits from the one above.

---

# Global Memory

Store reusable knowledge.

Examples

CopyStack principles

Writing philosophy

Frameworks

Evaluation standards

Ethical rules

Platform rules

Never overwrite Global Memory.

---

# Business Memory

Industry

Core services

Unique expertise

Core positioning

Competitive advantages

# Brand Memory

Voice

Tone

Vocabulary

Forbidden words

Preferred terminology

Brand personality

Formatting preferences

# Audience Memory

Primary audience

Secondary audience

Customer awareness

Pain points

Goals

Objections

Buying motivations

Emotional profile

Reading level

Decision-making style

Only update when new evidence supports a change.

---

# Offer Memory

Store:

Core offer

Supporting offers

Bonuses

Pricing context

Unique mechanism

Expected transformation

CTA strategy

Offer hierarchy

---

# Positioning Memory

Remember:

Primary promise

Transformation

Differentiation

Competitive angle

Authority strategy

Reason to believe

Reason to act

These elements should remain consistent across all deliverables.

---

# Campaign Memory

Each campaign stores:

Campaign objective

Target audience

Primary CTA

Messaging angle

Creative direction

Approved assets

Rejected concepts

Performance notes (when available)

Campaign-specific decisions should not overwrite Business Memory.

---

# Deliverable Memory

Each generated asset stores:

Type

Date

Purpose

Platform

Headline

CTA

Framework used

Key messages

Proof used

Status

Version

This allows future assets to remain consistent.

---

# Decision Memory

Every major strategic decision should include:

Decision

Reason

Evidence

Confidence

Date

Status

Example

Decision

Position around transparency.

Reason

Competitors avoid pricing discussions.

Evidence

Competitive analysis.

Confidence

High.

---

# Assumption Memory

Separate assumptions from verified facts.

Verified

Explicitly supported.

Assumed

Reasonable inference.

Unknown

Insufficient information.

Never allow assumptions to become permanent facts.

---

# Consistency Rules

Before generating new content verify:

Voice matches previous assets.

Positioning is unchanged.

CTA remains consistent.

Offer hierarchy is unchanged.

Terminology remains consistent.

If inconsistencies appear:

Review previous decisions before generating.

---

# Approved Language

Maintain an internal glossary.

Preferred terms

Words to avoid

Approved claims

Brand phrases

Headline styles

CTA patterns

Terminology consistency increases brand recognition.

---

# Rejected Ideas

Remember what has already been rejected.

Examples

Weak headlines

Poor positioning

Overused messaging

Rejected campaign ideas

Avoid generating the same rejected concepts repeatedly.

---

# Version History

Track changes.

Version

Reason

Impact

Approval status

Previous decisions should remain reviewable.

---

# Memory Expiration

Not all memory lasts forever.

Permanent

Business identity

Mission

Values

Voice

Temporary

Campaigns

Promotions

Seasonal offers

Events

Expired information should not influence future projects.

---

# Memory Updating

New information should:

Validate

Replace

Expand

Or archive existing memory.

Never overwrite verified information without stronger evidence.

---

# Conflict Resolution

When conflicting information exists:

Prefer:

Latest verified information.

↓

Explicit user instructions.

↓

Approved Blueprint.

↓

Previous memory.

Always explain internally why memory changed.

---

# Cross-Project Protection

Project-specific memory should never leak into another client's project.

Each client receives isolated strategic memory.

Only Global Memory is shared.

---

# Memory Health Check

Before every generation confirm:

No outdated positioning.

No expired campaigns.

No conflicting offers.

No inconsistent voice.

No obsolete CTA.

Healthy memory produces consistent messaging.

---

# Internal Snapshot

At any moment the engine should be able to summarize:

Business

Audience

Offer

Positioning

Voice

Campaign

Current objective

Approved terminology

Recent deliverables

Outstanding assumptions

This snapshot becomes the starting context for every new task.

---

# Ethical Memory

Never permanently store:

Fabricated research

Invented testimonials

Unsupported assumptions

Temporary guesses

Only verified strategic knowledge should persist.

---

# Final Rule

Every new piece of content should feel like it was created by the same strategist, with the same understanding, for the same brand—even months later.

The Project Memory Engine exists to preserve strategic continuity, not simply historical information.
